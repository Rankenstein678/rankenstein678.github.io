Datenbanken

Datenbank:

Eine Datenstruktur in der Daten gespeichert werden

Datenbankverwaltungssystem (DBMS):

Mithilfe eines Datebankverwaltungssystem kann eine Datenbank verwaltet und teils auch graphisch bearbeitet werden.

Datenbankschnittstelle:

Über eine Schnittstelle interagieren Programme mit der Datenbank. Diese Interaktion läuft über die SQL ab.

Begriffe

Entitätstyp

Ein Entitätstyp ist eine Beschreibung einer bestimmten Menge an Entitäten

Entität

Ein eindeutig identifizierbares Informationsobjekt. In der Datenbank ein Datensatz einer Tabelle

Attribut

Eine Eigenschaft einer Entität. Sie wird meist als Spalte einer Tabelle dargestellt.

Relation

Eine logische Verbindung zwischen Entitäten.

Kardinalität

Das Verhältnis einer Relation zwischen Entitäten.
1:n Eine Entität A ist mit n Entitäten B verbunden
m:n M Entitäten A sind mit n Entitäten B verbunden.

Tabelle

Sammlung verwandter Daten, die in einem strukturierten Format in einer Datenbank gespeichert sind. Sie stellt eine

Datensatz

eine Gruppe von inhaltlich zusammenhängenden Datenfeldern

Datenfeld

kleinste Einheit eines Datensatzes (ein Feld)

Schlüssel

Ein Wert der Entitäten in Datebanken eindeutig identiiziert

Primärschlüssel

Ein Feld / eine Gruppe von Feldern, die einen Datensatz eindeutig identifizieren

Fremdschlüssel

Feld , das tabellenübergreifend genutzt wird, um eine Verknüpfung zwischen verwandten Daten herzustellen

Modellierung

Vierecke stehen für Entitätstypen, Kreise für Attribute und Rauten für Relationen

ERM

UML

Normalformen

1. Normalform

Alle Attribute atomar + eindeutiger Primärschlüssel

2. Normalform

Jedes Nichtschlüsselattribut vom gesamten Primärschlüssel voll funktional abhängig

3. Normalform

Kein Nichtschlüsselattribut ist von einem Schlüsselattribut transitiv abhängig / kein Nichtschlüsselattribut hängt von einem anderen Nichtschlüsselattribut ab.
